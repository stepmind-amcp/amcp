# Contributing to AMCP / 贡献指南

Thank you for your interest in contributing to AMCP!  
感谢你对 AMCP 的贡献兴趣！

---

## Ways to Contribute / 贡献方式

### 1. Report Issues / 报告问题
Open an issue for: / 以下情况请开 issue：
- Bugs or ambiguities in the specification / 规范中的缺陷或歧义
- Missing examples or unclear explanations / 缺失示例或不清晰的说明
- Conformance test edge cases / 一致性测试边界情况

### 2. Improve Documentation / 改进文档
- Fix typos and grammar / 修复拼写和语法
- Add examples / 添加示例
- Translate to new languages / 翻译到新语言
- Write tutorials / 编写教程

### 3. Implement AMCP / 实现 AMCP
Build an AMCP-compliant node in any language:  
用任意语言构建 AMCP 兼容节点：
1. Study [`spec/conformance/level-1.md`](spec/conformance/level-1.md)
2. Implement the 8 Level-1 requirements / 实现8项 Level-1 要求
3. Run self-tests and document results / 运行自测并记录结果
4. Open a PR or issue linking your repo / 开 PR 或 issue 链接你的仓库
5. Register at [`stepmind/amcp-registry`](https://github.com/stepmind/amcp-registry)

### 4. Propose Protocol Extensions / 提议协议扩展
Via the RFC process: / 通过 RFC 流程：
1. Open a Discussion with tag `[RFC Idea]` / 开一个标记 `[RFC Idea]` 的 Discussion
2. Gather community feedback (≥7 days) / 收集社区反馈（≥7天）
3. Submit a formal RFC to [`rfcs/`](rfcs/) via Pull Request / 通过 PR 提交正式 RFC 到 `rfcs/`
4. TSC reviews and votes / TSC 审查并投票
5. Merged RFCs are scheduled for the next protocol version / 合并的 RFC 排期进入下一版本

---

## Pull Request Process / PR 流程

```
1. Fork the repo / Fork 仓库
2. Create a branch: feat/your-feature or fix/your-fix
3. Make changes / 进行修改
4. Ensure all existing tests pass / 确保现有测试通过
5. Open PR with clear description (zh + en both welcome) / 开 PR，中英文描述均欢迎
6. A maintainer will review within 7 days / 维护者将在7天内审查
```

## Code of Conduct / 行为准则

Be respectful, constructive, and inclusive.  
保持尊重、建设性和包容性。

We follow the [Contributor Covenant](https://www.contributor-covenant.org/).  
我们遵循[贡献者公约](https://www.contributor-covenant.org/)。

在 [Discussions](https://github.com/stepmind/amcp/discussions) 发帖 — 没有问题太小。
